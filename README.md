# LandauVPN

**LandauVPN** — десктопный менеджер VPN-профилей на Python с интерфейсом на `customtkinter`.

Приложение рассчитано на администратора: в нём есть вход по паролю, управление собственными VPN-профилями, отдельный пул бесплатных серверов из `vpn_servers.json`, быстрый запуск VPN и фоновая работа без зависания интерфейса.

## Возможности

- вход только под `admin`
- хранение пароля администратора локально в `~/.LandauVPN/master_config.json`
- управление собственными профилями OpenVPN / WireGuard / VPNGate
- отдельный JSON-файл со списком бесплатных серверов: `vpn_servers.json`
- обновление бесплатного пула из публичного VPNGate API
- поиск по списку профилей и бесплатных серверов
- двойной клик по серверу для подключения
- подключение выбранного профиля к VPN на уровне всего ПК
- отключение VPN из интерфейса
- фоновая загрузка и фоновое подключение без блокировки GUI
- журнал действий внизу окна

## Требования

- Python 3.10+  
- `customtkinter`  
- `requests`
- установленный клиент VPN:
  - **OpenVPN** для `.ovpn`
  - **WireGuard** для `.conf`

### Установка зависимостей

```bash
pip install customtkinter requests
```

## Запуск

```bash
python landau_admin_vpn_manager_ux.py
```

Если собираешь `.exe`, файл `vpn_servers.json` должен лежать рядом с программой.

## Вход в админ-панель

- логин: `admin`
- пароль по умолчанию: `Konstruk.tor.16.`

Пароль можно поменять во вкладке **Настройки**.

## Работа с бесплатными серверами

Файл `vpn_servers.json` хранит отдельный список бесплатных VPN-профилей. Приложение:

1. загружает его при старте,
2. показывает его во вкладке **Бесплатные VPN**,
3. может обновить список из публичного VPNGate,
4. сохраняет свежую версию обратно в JSON.

Каждый профиль в JSON имеет формат:

```json
{
  "name": "VPNGate Japan #1 — public-vpn-123.opengw.net",
  "kind": "vpngate",
  "source": "vpngate://public-vpn-123.opengw.net|219.100.37.123",
  "local_path": "",
  "enabled": true,
  "note": "score 123456, ping 18 ms, speed 24567",
  "country": "Japan",
  "ip": "219.100.37.123"
}
```

## Добавление собственного профиля

Во вкладке **Добавить** можно добавить:

- локальный файл `.ovpn` или `.conf`
- ссылку `http(s)://...` на конфиг
- VPNGate-формат: `vpngate://hostname|ip`

Пример:

```text
vpngate://public-vpn-257.opengw.net|219.100.37.208
```

## Как подключается VPN

Приложение запускает стандартный клиент:

- `openvpn --config <file>` для OpenVPN
- `wg-quick up <file>` для WireGuard на Linux/macOS
- соответствующий клиент/служба для Windows

Это означает, что программа подключает **сам ПК**, а не только отдельное приложение внутри Python.

## Удобства интерфейса

- поиск по профилям и бесплатным серверам
- двойной клик для быстрого подключения
- отдельная карточка с выбранным сервером
- кнопка для копирования `source`
- кнопка для открытия папки программы
- кнопка для обновления бесплатного пула
- параллельная проверка части профилей

## Использование

1. Запусти программу.
2. Введи пароль администратора.
3. Во вкладке **Бесплатные VPN** выбери сервер или нажми **Лучший бесплатный VPN**.
4. Во вкладке **Профили** можно подключить свой конфиг.
5. Во вкладке **Настройки** можно сменить пароль и обновить бесплатный пул.

## Troubleshooting

### VPN не запускается

Проверь, что:

- OpenVPN / WireGuard установлен в системе
- конфиг корректный
- для Linux доступны права `sudo`
- файл конфига существует по указанному пути

### Бесплатные серверы не обновляются

- проверь интернет
- проверь доступность VPNGate
- нажми **Перечитать JSON** или **Обновить из VPNGate**

### Интерфейс не показывает список

Проверь, что рядом с программой лежит `vpn_servers.json`. Если файла нет, обнови список из VPNGate.

## Безопасность

- пароль администратора хранится локально в хэше SHA-256
- встроенный пул бесплатных серверов — публичный и может меняться
- не используй неизвестные конфиги из ненадёжных источников

## Примечания

- VPNGate — публичный ресурс, у серверов может быть ограниченное время жизни.
- В Linux для `openvpn` и `wg-quick` могут понадобиться права root.
- Программа работает в фоне и не должна зависать при обновлении списков или подключении.
